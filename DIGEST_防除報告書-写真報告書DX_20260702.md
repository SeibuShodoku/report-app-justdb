## 確定した決定事項（時系列）

**① アーキテクチャ方針（確定）**
- **JUST.DB** = 事務データの源泉（システム・オブ・レコード）
- **WEBアプリ** = 作成・AI・出力の層（JUST.DBのGUI依存を排除）
- **Google Drive** = 正本（PDF＋再編集用JSON・管理番号で紐づけ）
- **JUST.DBへの書き戻し** = 限定フィールドのみ（金額・回数・薬剤・要約）
- **Supabase** = JUST.DB薬剤資材マスタのミラー（毎回JUST.DB APIを叩かない）

**② 報告書様式（確定）**
- 原本は**紺谷V**（防除作業完了報告書・城東支店・法令備付帳簿用）をベースに採用
- `/mock` タブで紺谷V・写真報告書・融合を切り替えて「議論のまな板」として提示できる状態を維持する方針

**③ Drive認証方式（確定・重要な経緯あり）**
- 外部SA（`report-drive-reader@seibot-proxy.iam.gserviceaccount.com`）を試みたが **Workspace の外部プリンシパルにはフォルダ子ファイルへの権限が継承されない**ことが判明して削除
- **社内ユーザーOAuth（refresh_token方式）** に切替。`mgmt-strat@seibu-s.co.jp` の `drive.readonly` トークン。OAuthクライアントは `seibu-dispatch-poc-tky`（dispatch-app 流用）
- Drive API を `seibu-dispatch-poc-tky` で有効化済

**④ VMのClaude Code認証（確定）**
- VM上のClaudeは一度 `ishibashi@seibu-s.co.jp`（個人アカウント）で認証されていたが、`mgmt-strat@seibu-s.co.jp`（Seibu-s Co 組織・Team・owner）に切替済
- D-AIDATAの「Team/API必須」要件を満たす

**⑤ Phase2 AIワーカー方式（確定）**
- VM上のClaude Codeをヘッドレス起動（`claude -p`）してreport.jsonを生成
- 追加APIキー不要・Team枠内・VMはGoogle資格情報を持たない

**⑥ ホスティング（確定）**
- Vercel Hobby は**商用利用不可**のため除外
- **Cloud Run（`seibu-dispatch-poc-tky`）** に決定（visit-planner・seibot-proxy と同一スタック）

**⑦ データモデル（確定）**
- 薬剤は横持ち（薬剤1・薬剤2…列）ではなく**縦持ち明細**（施工×薬剤×使用量×使用場所）
- 薬剤マスタは上位テーブルとして独立させ、施工明細と在庫の両方から参照

**⑧ D-AIDATA決定（dev-guidesリポに起票済）**
- 顧客写真をAIに渡す条件：Team/API（商用規約）必須・転送はBFF経由・写真はDriveにとどまる
