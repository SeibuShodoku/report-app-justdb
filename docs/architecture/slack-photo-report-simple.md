# 写真報告書システム（簡易版）

最終更新：2026-06-19
案件の写真と情報から、**AI が報告書の下書き**を作り、**人が WEB で仕上げる**（赤丸・微修正・版管理）。
（詳細版：`slack-photo-report-architecture.md`）

## 流れ（4ステップ）

```
[ 1. 入口（Slack） ]
   トピックの「📸報告書」ボタン → スレッドに写真用フォルダ（訪問ごとに分かれる）
   現場が写真をそのフォルダへ → 「📝報告書作成」を押す
        |
        v
[ 2. AIが下書き ]   ※VM常駐
   写真＋文脈（案件フォルダの書類・Slack要約）を読んで、各写真の見出し・所見・全体要約を生成
        |
        v
[ 3. 人が仕上げ ]   ※WEB・社内限定(@seibu-s.co.jp ログイン)
   赤丸・微修正（写真は触らず重ね描き）。保存すると版が残り、いつでもロールバックできる
        |
        v
[ 4. 保管・配布 ]
   最新版はリンクで常に開ける。紙/PDFが要る人はブラウザから任意で印刷
```

- **写真は無垢のまま**（赤丸はデータとして写真に重ねるだけ）。**最新版＝システムが保持／過去版＝案件フォルダの `_ai/` に履歴**。
- **PDFはシステムの仕事ではない**：欲しい人が印刷する。複数の報告書をまとめて管理したいのも人の運用。

## 精度のカギ

**写真だけで判断しない。** 案件フォルダの書類と Slack の会話を AI が読み、
「何の作業か（例：ネズミ駆除か防鳥か）」を文脈で正しく押さえてから書く。

## 使う部品（ざっくり）

| 部品 | 役目 |
|---|---|
| Slack 案件スレッド | 入口・会話の文脈 |
| Google Drive 案件フォルダ | 写真・書類の置き場 |
| AI ワーカー（VM常駐） | 下書きを自動生成 |
| WEB（社内限定） | 人が仕上げ・版管理（赤丸/微修正/ロールバック） |

## 状態

- 済：WEB単体（写真プリフィル＋印刷） / Slack抜きの自動生成（AIが下書き） / **Slackトリガー入口（prod・テスター限定）**
- 次：写真サブフォルダ化（訪問分離） / 完了返信（スレへ最新URL） / 版管理・注記UI / 案件ダイジェスト生成（文脈の質を上げる）
