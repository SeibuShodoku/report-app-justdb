# 写真報告書システム（簡易版）

最終更新：2026-06-19
案件の写真と情報から、**AI が報告書の下書き**を作り、**人が仕上げて PDF** にする。
（詳細版：`slack-photo-report-architecture.md`）

## 流れ（4ステップ）

```
[ 1. 入力 ]
   現場写真 ＋ 案件の文脈
   （Slack案件スレッド / Driveの書類：調査報告書・見積など）
        |
        v
[ 2. AIが下書き ]   ※VM常駐
   写真＋文脈を読んで、各写真の見出し・所見・全体要約を生成
        |
        v
[ 3. 人が仕上げ ]   ※WEB・社内限定(@seibu-s.co.jp ログイン)
   赤丸・微修正（AIの下書きを直すだけなので速い）
        |
        v
[ 4. PDFで保存 ]
   完成した写真報告書を PDF 化し、案件フォルダに保管
```

## 精度のカギ

**写真だけで判断しない。** 案件フォルダの書類と Slack の会話を AI が読み、
「何の作業か（例：ネズミ駆除か防鳥か）」を文脈で正しく押さえてから書く。

## 使う部品（ざっくり）

| 部品 | 役目 |
|---|---|
| Slack 案件スレッド | 入口・会話の文脈 |
| Google Drive 案件フォルダ | 写真・書類の置き場 |
| AI ワーカー（VM常駐） | 下書きを自動生成 |
| WEB（社内限定） | 人が仕上げ・PDF出力 |

## 状態

- 済：WEB単体（写真プリフィル＋PDF） / Slack抜きの自動生成（AIが下書き）
- 次：案件ダイジェスト統合（文脈の質を上げる） / Slackからのトリガー
