# Vercel + Drive運用方針（v0.1）

## 1. 基本構成

- 実行基盤: Vercel
- ファイル保存: Google Drive（既存の案件フォルダ）
- 起動元: JUST.DB（URLパラメータ連携）

## 2. コスト最適化

- Vercel無料枠を基準に開始
- PDF生成は明示操作時のみ実行
- 画像圧縮後に保存
- 不要な再出力を避ける

## 3. Drive保存方式

- 既定: 方式B
- 方式B: 帳票に割り当てた写真のみ + 設定ファイル + 生成PDF を保存
- 方式Aはオプションとして将来切替可能

## 4. 保存対象

- `report-{id}.pdf`
- `report-config-{id}.json`
- `photo-*.jpg`（帳票使用分のみ）

## 5. エラーハンドリング

- Drive保存失敗時はAPIで失敗を返し、リトライ可能な状態を保持
- 部分成功時は状態を記録し、再実行できる設計とする
