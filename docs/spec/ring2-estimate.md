# 見積書（リング2）仕様・現況

> **位置づけ**：案件ポータル北極星 `vision/case-portal.md §9` の **リング2＝見積書**。
> 見積書は報告書のサブ種別ではなく、案件ライフサイクル上 **見積／報告／請求の兄弟**。URL は `/estimate`（将来 `/invoice` 同階層）。
> 社内専用（Cloud Run＋IAP）。本書は計画＋現況。**JUST.DB の計算式の正本は別リポの `docs/justdb-dataflows.md`「見積系（tojo一族）」**（全式表＋Mermaid）。

## 0. 何を作っているか

JUST.DB の見積（GUI が遅く様式がバラつく）を、**入力を軽く・出力を固定**する WEB 作成面へ。サイトが**作成面の主役**、正本は将来 JUST.DB（会計）側で、**JUST.DB への書き戻しは別軸・後・API予算gate内**（報告書サイトと同じく当面 read-only 志向）。

- **見積は品目カタログでも自由入力でもなく、パラメトリックな原価積算（コストビルドアップ）**。物理量（薬剤・使用量・施工時間・人数・移動距離・坪数・回数）から原価を積み上げ、**原価係数**（＝目標原価率）で標準価格を逆算する。

## 1. 構造（親子3層）

```
見積（表紙）
  └─ 明細（施工内容・労務・移動・坪・回数・原価係数 … 1見積に複数）
       └─ 薬剤明細（縦持ちサブテーブル ＝ 1明細に複数）
```
- 表紙：見積日・顧客名・施工先名・件名・**移動距離（基本情報＝全明細に適用）**・支払条件・敬称・摘要・免責事項・合計。
- 明細：計算方式（一般／シロアリ坪単価）・業務タイプ・施工内容・労務（時間/人数）・回数・床下高所特殊・報告書作成費用・原価係数・見積金額（手入力＝標準価格上書き可）・値引。
- 薬剤明細：中分類→薬剤（販売価格表から）→使用量。複数可。

## 2. 計算式（正本＝justdb-dataflows.md・実CSV14行で検算一致）

定数は計算式設定（版）から。下記は「2026年度06月」版：人件費単価5800／移動単価270／労安費率0.16／諸経費率0.20／薬剤係数1.6／消費税0.10／原価係数{0.3, 0.55}。`ROUND`＝四捨五入（0.5切上）、`ROUNDUP(,百円)`＝百円切上。

**明細（一般施工金額計算）**
```
薬剤明細     = ROUND(薬剤単価[売価] × 使用量)        ※販売価格表は「売価」を持つ
薬剤売価_集計 = Σ 薬剤明細（明細フィールド合計・回数前）
薬剤売価_積算 = 薬剤売価_集計 × 作業回数
薬剤原価_積算 = ROUND(薬剤売価_積算 ÷ 薬剤係数)        ※売価を合計してから1回丸め（掛率グループごと）
施工人件費   = ROUND(人数 × 回数 × 施工時間変換 × 人件費単価)   ※施工時間変換＝時間を2桁丸め。割増は含めない
割増分施工人件費 = ROUNDUP(施工人件費 × (割増料金 − 1), 百円)    ※施工コストには入れない別建て
移動コスト   = 移動単価 × 人数 × 回数 × 移動距離           ※×人数が肝（ID66=270×4×4×150=648000）
施工コスト   = 施工人件費 + 移動コスト + 報告書作成費用 + 薬剤原価_積算
諸経費       = ROUNDUP(施工コスト × 諸経費率, 百円)
労働安全衛生費 = ROUNDUP(労安費率 × 床下高所特殊係数 × 施工コスト, 百円)
標準価格     = ROUND(施工コスト ÷ 原価係数)               ← 見積金額の既定（手調整可）
標準価格(薬剤吸収) = ROUND((施工コスト − 薬剤原価) ÷ 原価係数) + 薬剤売価_積算
                                                     ※薬剤は売価のまま乗せ、施工マージンは労務等だけに掛ける版
```

**明細（シロアリ坪単価計算）**
```
見積金額   = 坪単価 × 坪数                ※売価は坪で決まる（薬剤明細とは別フィールド）
防蟻剤売価 = ROUND(防蟻剤坪単価 × 坪数) → 防蟻剤原価 = ROUND(防蟻剤売価 ÷ 薬剤係数) → 施工コストへ
```

**共通**
```
粗利額 = 見積金額 − 施工コスト ; 粗利率 = ROUND(粗利額 ÷ 見積金額, 3)
経費控除後見積金額 = 見積金額 − 移動コスト − 薬剤売価_積算 − 報告書作成費用 − 諸経費 − 割増分施工人件費 − 労働安全衛生費 + 値引額
   ＝「材料(薬剤売価)・経費を抜いた後に工賃＋利益として残る額」の診断指標（最終価格には非関与）
表紙：本体 = Σ 値引後見積金額 ; 消費税 = ROUND(本体 × 消費税率) ; 税込 = 本体 + 消費税
```

**床下高所特殊係数**：床下/高所/特殊作業**あり=1（労安フル16%）／なし(基本)=0.1（1.6%）**。労安は標準価格には入らず、経費控除後にだけ効く（会社が負う安全衛生コストの可視化）。

## 3. シロアリ施工プラン（坪単価1択 → 防蟻剤/原価を自動）

計算方式＝シロアリ坪単価 で「施工プラン（坪単価）」を1つ選ぶと、見積坪単価・防蟻剤・防蟻剤売価/坪が自動セット（手上書き可）。業務タイプ シロアリ新築→新築のみ／シロアリ既築→木部＋土壌 に絞る。

| 防蟻剤(売価/坪) | 木部＋土壌(既築) | 新築木部のみ | 新築土壌込 |
|---|---|---|---|
| エディクラ 1611 | 8000 | 2400 | 3400 |
| オプティガード 1768 | 8240 | 2500 | 3500 |
| タケロック 1989 | 8800 | 3000 | 4000 |
| 天然ピレトリンMC 3162 | 11000 | 3600 | 4600 |

（坪単価が施工種別＋防蟻剤を一意に決める。防蟻剤掛率＝1.6。エディタ `TERMITE_PLANS`/`TERMITE_CHEMS` にハードコード＝将来は版管理/マスタ化候補）

## 4. データ層

- **計算式設定**＝`estimate_settings`（migration `20260624150000`）：`label`=識別キー(版・例 "2026年度06月")＋`effective_from`＋各単価/率＋`is_active`(論理削除)・RLS。見積日に有効な最新版を `resolveSettingsForDate` で解決＝**過去見積の再現性**。管理画面 `/admin/estimate-settings`。
- **販売価格表（薬剤資材マスタ）**＝`chemical_products`（migration `20260624160000`＋`170000` で `sort_order`）：JUST.DB マスタの単一ミラー。`price_table_id` PK・売価/原価/`markup`(品目別1.6/3.2)/単位/`category`(中分類)/`report_name`/`is_active`。**見積の価格と報告書の表示名/カテゴリを1ソース供給**。
  - 取り込み＝`scripts/import-price-book.mjs`（Shift-JIS CSV→`TextDecoder`→最小CSVパーサ→upsert。**価格はリポに焼かずランタイム取り込み**＝CSV は gitignore）。**表示順は CSV(JUST.DB)行順を `sort_order` で保持**（store は sort_order→品目名で JS ソート＝カラム未追加でも安全）。約426品目/12カテゴリ。
- 既存 `chemicals`（カスケード・本番稼働）とは粒度が別。今は触らず、JUST.DB 実連携時に統合。

## 5. エディタ（`/estimate`・`src/components/estimate-editor.tsx`）

- 純粋エンジン `src/lib/estimate-calc.ts` を**クライアントでも実行**（server依存なし）＝物理量入力で `calcEstimate` を useMemo ライブ計算。
- 薬剤・中分類・業務タイプ・防蟻剤は**検索つきセレクト**（`src/components/search-select.tsx`・426品目を入力絞り込み）。
- レイアウト：2/3カラム横並び（`.est-row-2/3`・モバイルもペア維持）。明細はカード（見出しに業務タイプ・金額・畳み）。**合計は sticky バーで常時下部追随**。
- **計算ステータス**（JUST.DB 照合用・全項目）＝既定折りたたみ・ⓘタップヒント（薬剤吸収/経費控除後/労安・床下/標準価格）。
- **原価係数の UI 表記＝粗利率%**（粗利率=1−原価係数。0.3→70%/0.55→45%。内部値は原価係数）。**粗利率の健全性カラー**（赤字=赤/<25%=橙/緑）。
- 報告書作成費用＝固定プルダウン（0/1000/2000/3000/4000/6000/8000＝作成時間目安）。割増料金＝係数プルダウン（1/1.25/1.5）。見積金額/値引額＝カンマ表示。

## 8. 未確定・留意

- **入力フィールドの素性**：単価・各率・薬剤係数・原価係数・坪単価の**マスタ値の根拠**は JUST.DB「データテーブル定義.xlsx」側の確認が要る（式中の意味は判明）。
- **旧版/NEW版の併存**：移行過渡期で旧 `tojo見積作成` と `(NEW)` が両稼働。どちらが発火するか（パネル紐付け）要確認。
- **シロアリ施工プラン／坪単価表**はエディタにハードコード＝版管理/マスタ化は将来。
- **JUST.DB への見積書き戻し**は別軸・後・API予算gate内（`open-issues.md §0`）。
- 複数薬剤で掛率が混在する場合の原価丸めは「掛率グループごとに売価合計→1回丸め」で実装（JUST.DB 単一係数前提の自然な一般化）。
