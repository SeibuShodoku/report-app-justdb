# 要件定義（v0.7）

## 1. 背景

既存のExcel帳票は、端末ごとの差異（フォント、プリンタードライバ、Excelバージョン）により印刷レイアウト崩れが発生する。

## 2. 目的

- 報告書をブラウザで作成できること
- 1件単位で固定レイアウトのPDFを出力できること
- JUST.DBから案件単位で起動できること
- 画像と設定を再利用して再編集・再出力できること

## 3. スコープ

### 3.1 対象

- 報告書作成画面
- 画像注記（マウス描画）
- PDF出力
- JUST.DB連携（URLパラメータ受け渡し）
- ローカル写真取込（単体/フォルダ一括）
- Googleドライブ保存
- 設定ファイル保存・読込（再編集）

### 3.2 非対象（現時点）

- CSV出力
- Excel最終帳票の正式運用
- 複数案件一括PDF出力
- JUST.DBへの報告書データ書き戻し

## 4. 決定事項

- 出力形式はPDFを正式採用する
- 出力単位は1件ずつとする
- ページ構成は「表紙 -> 写真報告ページ群 -> 最終詳細ページ」とする
- 写真報告ページは初期実装で8枚レイアウトを採用する
- JUST.DBはアプリURLを保持し、IDをURLパラメータで渡す
- 受信したIDはアプリ側データの外部キーとして保持する
- JUST.DBへの書き戻しは行わない
- Excel出力は必須要件にしない
- 写真はブラウザからローカル選択で取り込む
- フォルダ単位の一括取込を可能にする
- 実行基盤はVercelを採用する
- 保存方式の既定値は方式B（帳票に割り当てた写真のみ保存）とする
- `driveFolderUrl` を正規入力とする
- 当面はトークンなし運用とする

## 5. 成果物定義

- 報告書データ（JSON）
- 注記付き画像データ
- 1件単位PDF
- 再編集用設定ファイル

## 6. 非機能要件（初期）

- 入力必須項目のバリデーション
- 監査ログ（作成者、作成日時、案件ID）の記録
- コスト最適化（Vercelの無料枠を優先利用）
