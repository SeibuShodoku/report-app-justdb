# 報告書・PDF仕様（v0.3）

## 1. 出力方針

- 正式提出物はPDFとする
- Excelは正式提出フォーマットから除外する
- 出力単位は1件単位に限定する

## 2. ページ構成（確定）

### 2.1 1ページ目: 表紙ページ

- タイトル
- 案件キー情報（案件ID、調査予定ID、施工予定ID）
- ヘッダー情報（作成者、作成日、報告区分）
- 見出し写真（任意）

### 2.2 中間ページ: 写真報告ページ

- 1ページに複数写真を配置する
- 各写真に見出し（キャプション）を設定可能
- 各写真に注記（マウス描画）を設定可能

### 2.3 最終ページ: 詳細報告ページ

- テキスト中心ページ
- 所見、実施内容、対応結果、次アクションを記載

## 3. 写真取込仕様

- 単体取込: ファイル選択で写真を取り込む
- 一括取込: フォルダ選択で複数写真を取り込む
- 取り込んだ写真一覧から、8枚レイアウト枠へ割り当てる

## 4. 写真ページレイアウト

- 初期実装は `8枚レイアウト` を標準とする
- 将来拡張で `4枚レイアウト` を追加可能にする
- ページをまたぐ写真配置は自動分割する

## 5. 画像注記機能（最小仕様）

- 対応ツール: フリーハンド、直線、矢印、テキスト
- 対応操作: 取り消し（Undo）、やり直し（Redo）
- 写真ごとに注記データを保持する

## 6. Googleドライブ保存仕様

保存方式は以下2案のいずれかを選択可能とする。

- 方式A: 取り込んだ全写真 + 設定ファイル + 生成PDFを保存する
- 方式B: 帳票へ割り当てた写真のみ + 設定ファイル + 生成PDFを保存する

## 7. 設定ファイル仕様

- 報告書の入力値、写真割り当て、注記データ、出力設定を保存する
- 設定ファイル読込時にGoogleドライブ上の写真と設定を復元し、再編集可能にする
- 再編集後に再出力（PDF）を可能にする

## 8. PDF要件

- A4固定レイアウト
- 端末依存を避けるため、サーバー側で生成する
- フォント埋め込みに対応する
- ページ順序は「表紙 -> 写真ページ群 -> 最終ページ」に固定する

## 9. 実装上の原則

- PDF生成ロジックはサーバー機能として分離する
- レイアウトはテンプレート化し、コード直書きを避ける
- 注記再編集のため、画像と注記データを分離保存する
