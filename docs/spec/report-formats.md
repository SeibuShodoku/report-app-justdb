# 報告書フォーマット仕様（v1.0）

対象は西武消毒・城東支店の**防除作業報告書**。原本は `docs/reference/防除作業報告書-原本-2006.xlsx`。

## 1. 方針

- 同一の入力データから、**タブ切替で複数様式を出力**する（紺谷V / 写真報告書 / 融合）。
- これは「統合の手前」の設計。各様式を安く・並べて出せるようにし、議論のまな板を作る。
  詳細は `docs/architecture/overview.md`。
- 正式提出はPDF（A4固定）。出力単位は1件（1施工）。
- モック実装: `/mock`（`src/app/mock/page.tsx`）。

## 2. 紺谷V（防除作業完了報告書）

法令（ビル管理法・食品衛生法・労働安全衛生法）の**備付帳簿用**。お客様控。

構成（原本準拠）:

1. 社判ロゴ・標題「防除作業完了報告書」・「お客様控」
2. 宛先（◯◯御中）＋報告日
3. 管理責任者 / 作業責任者 / 作業員
4. 挨拶文
5. 施工日時（日付＋作業時間 from〜to）／施工場所
6. **施工内容テーブル（縦持ち・複数行）**: 対象害虫・使用薬剤・処理方法・薬剤使用量・備考
7. **駆除作業報告（自由文）**: 施工内容テーブル＋定型文から**自動生成**し編集可
   （「以上、施工完了しました。」で締める）
8. 生息状況テーブル: 対象・状況
9. 効果判定: `- / + / ++ / +++`
10. 結語「上記の通り実施いたしました。」

## 3. 写真報告書

- 写真をローカルから取込（単体／フォルダ一括）、見出し・注記つきで配置。
- 各写真に**赤丸等の注記**（指摘箇所マーキング）。※現状モックは未実装（次イテレーション）。
- 顧客・施工先・施工日の見出しは紺谷Vと共有。

## 4. 融合

- 紺谷Vの完了報告 ＋ 写真報告を1セットで出力。
- 顧客により紺谷V単体／写真報告書単体／融合を選べることを想定。
- レイアウトは初期案（連結）。あるべき融合形は要議論（`open-issues`）。

## 5. 入力と保存の分離（重要）

- 保存の正解は正規化（縦持ち）だが、入力で行をN件作るのはモバイルで非直感的。
- WEB層が**入力（直感的）↔保存（正規化）の変換**を担う。
  入力は「行を足す／前回・見積から事前充填して確認・微調整」。
- 施工内容の対象害虫→使用薬剤→処理方法は**カスケード**（JUST.DB薬剤資材のSupabaseミラー）。
  詳細は `docs/architecture/justdb-supabase-integration.md`。

## 6. 写真取込

- 単体取込（ファイル選択）／一括取込（フォルダ選択）。
- 保存前に**画像圧縮**（JUST.DB直投入の容量オーバー・作り直しを防ぐ）。

## 7. 設定ファイル（再編集）

- 入力値・写真割当・注記・出力設定を**再編集用JSON**として保持。
- DriveにPDF（清書）と再編集JSONの両方を、管理番号で紐づけて保存。
- 読込で復元し再出力できる（PDFだけだと再編集不可のため両方持つ）。

## 8. PDF要件

- A4固定。端末依存を避けるためサーバー側生成（フォント埋め込み）。
- ※モックは `window.print()` でPDF化（印刷CSSで操作系を除去）。本番のサーバー生成は別途。
